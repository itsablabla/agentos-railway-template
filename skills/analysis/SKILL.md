# Analysis Skill

Structured frameworks for analyzing data, situations, and decisions.

## When to Use This Skill

Use this skill when analyzing a dataset or business situation, comparing options and
making recommendations, identifying root causes of problems, or evaluating risks and opportunities.

## Analytical Frameworks

### MECE (Mutually Exclusive, Collectively Exhaustive)
Break down any problem into categories that do not overlap (mutually exclusive)
and cover everything (collectively exhaustive). Every customer should fall in exactly one segment.

### Root Cause Analysis (5 Whys)
Ask "Why?" five times to trace a problem to its root cause. For example:
Problem: Website conversion rate dropped 30% → Why: Fewer users completing checkout →
Why: New checkout form has more fields → Why: A/B test added required phone field →
Why: Marketing team added it without UX review → Root Cause: Missing UX review process.

### Decision Matrix
Compare options across multiple weighted criteria. Multiply each score by its weight,
sum the weighted scores, and select the highest-scoring option.

### SWOT Analysis
Analyze Strengths (internal positives), Weaknesses (internal negatives),
Opportunities (external positives), and Threats (external negatives).

### Data Analysis Process
1. Understand: What question are we answering?
2. Collect: Gather relevant data
3. Clean: Handle missing values, outliers, duplicates
4. Explore: Descriptive statistics, distributions, correlations
5. Model: Apply appropriate analysis technique
6. Interpret: What do the results mean?
7. Communicate: Present findings clearly

## Output Standards
Always state the question being answered. Show data sources and limitations.
Distinguish correlation from causation. Quantify uncertainty where possible.
Lead with the insight, not the methodology.
